Overview

CI/CD setup
eksctl create cluster --name trend-cluster --region ap-south-1 --nodegroup-name workers --node-type t3.medium --nodes 2 (to create two worker nodes to run the container)
Made GitHub repository as source

# Deployment.yml - Runs the application 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: trend-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: trend
  template:
    metadata:
      labels:
        app: trend
    spec:
      containers:
      - name: trend
        image: kesterr15/trend-app:latest
        ports:
        - containerPort: 80
Creates a Deployment named trend-app
Ensures 1 replica (1 pod always running)
Uses Docker image: kesterr15/trend-app:latest
Exposes container port 80 inside the pod
Pod is scheduled on a worker node

# service.yml - Gives external access
apiVersion: v1
kind: Service
metadata:
  name: trend-service
spec:
  type: LoadBalancer
  selector:
    app: trend
  ports:
    - port: 80
      targetPort: 80
Creates a Service named trend-service
Type: LoadBalancer → creates AWS ELB
Routes traffic to pods with label app: trend
Exposes port 80

now test app is working in browser

Create pipeline in Jenkins and run

Groovy script 
pipeline {
  agent any

  environment {
    DOCKERHUB_CREDENTIALS = 'dockerhub-creds'
  }

  stages {

    stage('Clone') {
      steps {
        git branch: 'main', url: 'https://github.com/kestersam30-stack/Project2.git'
      }
    }

    stage('Build Docker Image') {
      steps {
        sh 'docker build -t trend-app .'
      }
    }

    stage('Push to DockerHub') {
      steps {
        withCredentials([usernamePassword(credentialsId: 'dockerhub-creds', usernameVariable: 'USER', passwordVariable: 'PASS')]) {
          sh '''
          docker login -u $USER -p $PASS
          docker tag trend-app $USER/trend-app:latest
          docker push $USER/trend-app:latest
          '''
        }
      }
    }

    stage('Deploy to EKS') {
      steps {
        sh 'kubectl apply -f deployment.yml'
        sh 'kubectl apply -f service.yml'
      }
    }
  }
}

GitHub → Build Docker Image → Push to DockerHub → Deploy to EKS
